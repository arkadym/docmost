# OneNote / Joplin HTML Import

## Background

When migrating notes from OneNote to Docmost, a common workflow is:

1. Import the `.onepkg` file into **Joplin** (which reads the OneNote export)
2. Export from Joplin as **Markdown + HTML** ZIP — Joplin writes each note as a `.html` file

The resulting ZIP produces HTML files with two additional data sources beyond raw HTML content.

## File Structure

Each exported `.html` file has:

### 1. YAML Frontmatter (before `<!DOCTYPE HTML>`)

```
---
title: My Note Title
updated: 2026-03-30 10:44:37Z
created: 2026-03-30 10:44:36Z
---
```

### 2. Joplin Title Block (first element inside `<body>`)

```html
<div class="title" ...>
  <div class="container-outline">
    <!-- First container: note title -->
    <div class="outline-element">
      <span style="font-family: Calibri Light; font-size: 20pt;">My Note Title</span>
    </div>
  </div>
  <div class="container-outline">
    <!-- Second container: date and time (localized display strings) -->
    <div class="outline-element"><span>24 марта 2019 г.</span></div>
    <div class="outline-element"><span>14:05</span></div>
  </div>
</div>
```

This div is generated by OneNote and is decorative — it duplicates the title and shows a human-readable date. It should be **removed** from the imported content.

### 3. Main Content

Everything after the `.title` div is the actual note content.

## What We Import

### Timestamps (`createdAt` / `updatedAt`)

Parsed from frontmatter properties:

| Frontmatter key      | Maps to     |
|----------------------|-------------|
| `created`            | `createdAt` |
| `updated`, `modified`| `updatedAt` |

This applies to both `.html` and `.md` files that contain YAML frontmatter.

### Page Title

The title is resolved from multiple sources using the **longest wins** rule (since OneNote titles can be truncated in filenames/frontmatter):

1. **Body title span** — text of the first `<span>` inside `div.title > div.container-outline` (most complete, straight from OneNote)
2. **Frontmatter `title:`** — set by Joplin from metadata (may be truncated to filename limits)
3. **Filename** — existing fallback

If both body title and frontmatter title are present and one is a prefix of the other (e.g. filename got truncated), the longer one wins.

### Title Div Removal

The entire `<div class="title">` block is stripped from the HTML before conversion to ProseMirror, so it does not appear as content in the imported page.

## Implementation

**File**: `apps/server/src/integrations/import/services/import.service.ts`

- `processJoplinHtml(html)` — uses `cheerio` to detect, extract title from, and remove the `.title` div
- `.html` branch in `importPage` now:
  1. Calls `extractFrontmatter` to strip YAML and get `created`/`updated` dates
  2. Calls `processJoplinHtml` on the body HTML for title extraction and div removal
  3. Resolves final title as `longest(bodyTitle, frontmatterTitle) || fileName`
  4. Passes `createdAt`/`updatedAt` to `insertPage`
- `.md` branch similarly extracts dates from frontmatter and passes them to `insertPage`

## Known Limitation

Joplin has a bug where notebook (folder) names in Russian/Cyrillic are stored with wrong encoding (`cp1251` bytes interpreted as `UTF-8`) when importing from OneNote on Russian-locale Windows. This produces garbled folder names like `Р"РѕРєСѓРјРµРЅС‚С‹` for `Документы`. This is a **Joplin-side issue** and cannot be reliably fixed on import without false positives on legitimate text. Workaround: rename notebooks in Joplin before exporting.
